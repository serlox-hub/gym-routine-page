  <action>
    **Strategy:** For each consumer file, replace relative imports of shared utils with `@gym/shared` imports. Multiple relative imports from different shared utils in the same file should be consolidated into a single `import { ... } from '@gym/shared'` line.

    **Shared util module names to replace** (imports FROM these files change to @gym/shared):
    arrayUtils, bodyMeasurementCalculations, bodyWeightCalculations, calendarUtils, constants (PARTIAL — see below), dateUtils, measurementConstants, measurementTypes, routineExerciseForm, routineIO (PARTIAL — see below), routineTemplates, setUtils, supersetUtils, textUtils, timeUtils, validation, workoutCalculations, workoutTransforms

    **PARTIAL replacement rules:**
    - `constants.js`: Imports of shared exports (QUERY_KEYS, RIR_OPTIONS, RIR_LABELS, REST_TIME_*, RIR_MIN, RIR_MAX, SESSION_STATUS, SENSATION_LABELS, SENSATION_COLORS, getSensationColor, MUSCLE_GROUP_COLORS, getMuscleGroupColor) → `@gym/shared`. Imports of `getMuscleGroupBorderStyle` → keep as relative import from local `constants.js` stub.
    - `routineIO.js`: Imports of shared pure exports (ROUTINE_JSON_FORMAT, ROUTINE_JSON_RULES, buildChatbotPrompt, buildAdaptRoutinePrompt) → `@gym/shared`. Imports of supabase-dependent functions (exportRoutine, importRoutine, duplicateRoutine, downloadRoutineAsJson, readJsonFile) → keep as relative import from local `routineIO.js`.
    - If a file imports BOTH shared and platform-specific exports from the same module, it needs TWO import lines: one from `@gym/shared` and one from the relative path.

    **Import path patterns to match** (varying depth):
    - `from '../lib/FOO.js'` or `from '../lib/FOO'` (most common in components/hooks)
    - `from '../../lib/FOO.js'` or `from '../../lib/FOO'` (deeper subdirectories)
    - `from './lib/FOO.js'` or `from './lib/FOO'` (from within src/ directly)
    - `from '../../../lib/FOO.js'` (rare, check if exists)

    **Process for each file:**
    1. Identify all `import { ... } from '...lib/SHARED_MODULE'` lines
    2. Collect all named imports from shared modules
    3. Replace with single `import { name1, name2, ... } from '@gym/shared'`
    4. Keep any platform-specific imports (supabase, styles, videoStorage, queryClient, getMuscleGroupBorderStyle, routineIO supabase functions) as relative imports

    **Verification after replacement:**
    - `grep -rn "from.*\/lib\/arrayUtils" apps/` — should return 0 results
    - `grep -rn "from.*\/lib\/dateUtils" apps/` — should return 0 results
    - Repeat for all fully-shared modules
    - `grep -rn "from.*\/lib\/supabase" apps/` — should still have results (platform-specific)
    - `grep -rn "from.*\/lib\/styles" apps/` — should still have results
    - `grep -rn "from.*\/lib\/constants" apps/` — should only have getMuscleGroupBorderStyle imports
    - `grep -rn "from.*\/lib\/routineIO" apps/` — should only have supabase-dependent function imports

    **Do NOT update:**
    - Imports within `packages/shared/src/lib/` (sibling imports like `./textUtils.js` stay relative)
    - Imports of platform-specific files (supabase.js, styles.js, videoStorage.js, queryClient.js)
    - Test file imports within packages/shared (they test local files)

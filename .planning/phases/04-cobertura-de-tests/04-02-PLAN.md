Create `packages/shared/src/stores/createAuthStore.test.js` following the createWorkoutStore.test.js pattern exactly.

Setup:
- `makeSupabaseMock()` that returns `{ auth: { onAuthStateChange, getSession, signInWithPassword, signUp, signOut, resetPasswordForEmail } }` with sensible defaults (see 04-RESEARCH.md Pattern 3)
- `beforeEach` creates fresh `useAuthStore = createAuthStore(supabaseMock)` and optionally `supabaseMock = makeSupabaseMock()`

Test groups and cases:

**Initial state:**
- initial state has isLoading true, user null, error null
- isAuthenticated() returns false when no user

**initialize():**
- sets user from existing session (getSession returns session with user)
- sets isLoading false after initialize (no session case)
- registers onAuthStateChange listener
- calls onBeforeInitialize hook if provided
- handles getSession error gracefully (sets error, isLoading false)

**login():**
- sets user on successful signInWithPassword
- sets error on failed signInWithPassword
- clears previous error before attempting login

**signup():**
- succeeds and returns data (no auto-login, Supabase handles confirmation flow)
- sets error on failed signUp

**logout():**
- clears user and resets state
- calls onBeforeLogout hook if provided
- calls supabase.auth.signOut()

**resetPassword():**
- sets passwordRecovery on success
- calls onResetPasswordOptions hook if provided (for redirect URL)
- sets error on failure

**clearError() / clearPasswordRecovery():**
- clearError sets error to null
- clearPasswordRecovery sets passwordRecovery to null

**onAuthStateChange callback:**
- updates user when auth state changes to SIGNED_IN
- clears user when auth state changes to SIGNED_OUT

Use `act()` from `@testing-library/react` for state changes. Use `await act(async () => { await useAuthStore.getState().initialize() })` for async actions.

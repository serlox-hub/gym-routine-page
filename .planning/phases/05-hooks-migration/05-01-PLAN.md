   - Module-level `_authStore = null` and `_workoutStore = null`
   - `initStores({ authStore, workoutStore })` — sets both refs. Throw if called twice (defensive).
   - `useAuthStore(selector)` — calls `_authStore(selector)` if selector provided, else `_authStore()`. Throws if not initialized.
   - `useWorkoutStore(selector)` — same pattern with `_workoutStore`.
   - These are thin wrappers that delegate to the actual Zustand store hooks passed in via `initStores`.

2. Create `packages/shared/src/notifications.js` (per locked decision):
   - Module-level `_notifier = null`
   - `initNotifications(showToast)` — sets `_notifier = { show: showToast }`. Per locked decision, RN wires `Toast.show`, web wires nothing (no-op by default).
   - `getNotifier()` — returns `_notifier` (null if not initialized, callers use optional chaining: `getNotifier()?.show(msg, type)`).
   - Do NOT export `getNotifier` from the barrel — it is internal to shared hooks.

3. Check if `duplicateRoutine` can be moved to `packages/shared/src/api/routineApi.js`:
   - `duplicateRoutine` calls `exportRoutine` and `importRoutine` from the same `routineIO.js` file.
   - Both `exportRoutine` and `importRoutine` use the Supabase client directly via `import { supabase } from './supabase.js'`.
